Integration Documentation

1. Authentication

All API requests must include your API key in the x-api-key header.

x-api-key: YOUR_SECRET_API_KEY

2. Sending an OTP

To send an OTP, use the /api/send-otp endpoint. You must specify a templateId and optionally templateData for placeholders.

Dynamic Placeholders: In your template body/subject, use {{VAR_NAME}}. The service automatically handles {{OTP}}. 
// Javascript fetch() example
fetch('/api/send-otp', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'x-api-key': 'sk_your_key_here'
    },
    body: JSON.stringify({
        email: 'user@gmail.com',
        templateId: 'otp_verification', // The ID you created
        templateData: {
            USERNAME: 'John Doe',     // Replaces {{USERNAME}} in template
            APP_NAME: 'Sayban App'    // Replaces {{APP_NAME}} in template
        }
    })
})
.then(res => res.json())
.then(data => console.log(data));

3. Verifying an OTP

When the user enters the code, send it back to /api/verify-otp for validation. 

fetch('/api/verify-otp', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'x-api-key': 'sk_your_key_here'
    },
    body: JSON.stringify({
        email: 'user@gmail.com',
        code: '123456' // The 6 digit code
    })
})
.then(res => res.json())
.then(data => {
    if (data.message === 'OTP verified successfully') {
        // Successful verification!
    }
});

Pro Tip: No generation needed!

You don't need to generate the 6-digit code yourself. Our service handles generation, secure storage, and expiry (5 minutes) automatically!
